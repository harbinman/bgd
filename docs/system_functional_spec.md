# 《喵喵补光灯》系统功能规格说明书 (Detailed v2.1)

## 1. 产品概述
《喵喵补光灯》(MiaoMiao Fill Light) 是一款专为暗光摄影设计的移动辅助应用。通过将手机屏幕转化为可控的高亮度光源，并结合独特眼神光映射技术，为用户提供便携式的“专业影棚级”补光体验。

## 2. 核心功能模块详细说明 (12 宫格快捷工具)

基于 UI 截图中的 4x3 功能矩阵，系统需实现以下 12 个核心工具模块：

### 2.1 补光基础 (Lighting Basics)
1. **全屏 (Full Screen)**: 
   - **逻辑**: 一键隐藏所有 UI 控件（除返回手势外），使屏幕以设定的色温/颜色进行全覆盖发光。
2. **色卡 (Color Card)**: 
   - **交互**: 唤起底部滑动手册，支持快速切换 11+ 类摄影色温预设（冷白、暖阳、氛围蓝等）。
3. **亮度 (Brightness)**: 
   - **逻辑**: 直接调用硬件 API 调节屏幕背光，支持在 App 内通过垂直滑动即时微调亮度。

### 2.2 拍摄参数 (Shooting Parameters)
4. **比例 (Ratio)**: 
   - **选项**: 支持 9:16 (全面屏)、3:4 (经典人像)、1:1 (正方形社交图)。
   - **实现**: 动态调整 `CameraPreview` 的 `AspectRatio`。
5. **高清 (HD)**: 
   - **逻辑**: 切换预览分辨率（720P/1080P/4K），优化样片导出质量，支持高码率录制。
6. **缩放 (Zoom)**: 
   - **逻辑**: 支持单指滑动或双指捏合变焦，UI 实时显示变焦倍数（1x - 5x）。

### 2.3 拍摄模式 (Capture Modes)
7. **单拍 (Single Shot)**: 
   - **逻辑**: 默认快门模式，点击即抓拍当前画面。
   - **实施状态**: ✅ 已完成（里程碑 4，2026-04-13）
   - **功能特性**:
     - 点击快门按钮拍摄照片
     - 触觉反馈 (`HapticFeedback.mediumImpact()`)
     - 闪光动画效果（100ms 白色覆盖层）
     - 自动保存到系统相册
     - 支持应用滤镜后保存
     - 缩略图预览（快门按钮左侧）
   - **核心方法**: `_takePicture()` in `lighting_page.dart`

8. **延迟 (Delay)**: 
   - **配置**: 提供 3s/5s/10s 倒计时拍摄。屏幕中央需显示巨大的倒计时跳字。
   - **实施状态**: ✅ 已完成（里程碑 4，2026-04-13）
   - **功能特性**:
     - 定时器设置对话框（关闭/3s/5s/10s）
     - 全屏半透明黑色背景
     - 中心显示倒计时数字（120px 大字体）
     - 数字缩放动画（1.5 → 1.0）
     - 每秒轻触觉反馈
     - 倒计时结束自动拍摄
   - **核心方法**: `_startCountdown()` in `lighting_page.dart`

9. **自拍 (Selfie)**: 
   - **功能**: 切换前后摄像头，并默认开启暗光实时增强算法。
   - **实施状态**: ⏳ 部分完成（前置摄像头已默认启用，切换功能待实现）

**连拍模式 (Burst Mode)** [NEW - 里程碑 4]:
   - **实施状态**: ✅ 已完成（2026-04-13）
   - **功能特性**:
     - 长按快门按钮触发连拍
     - 连拍速度: 约 3.3 张/秒 (300ms 间隔)
     - 显示连拍计数器
     - 松开按钮停止连拍
     - 异步队列处理照片保存
   - **核心方法**: `_startBurstMode()`, `_burstCapture()` in `lighting_page.dart`

### 2.4 辅助工具 (Auxiliary Tools)
10. **音效 (Sound Effect)**: 
    - **逻辑**: 快门音效开关，静音模式下拍摄不发出物理快门声。
11. **爱心眼 (Heart Eyes)**: 
    - **原理**: 在黑场中央投影淡粉色心形轮廓。
    - **效果**: 通过光影反射在瞳孔中刻画出“心形眼神光”，并支持调节形状粗细。
12. **更多 (More)**:

13. **语言切换 (Language Switch)**: [NEW]

    - **功能**: 实现全局界面的中英文双语一键切换。

    - **位置**: 位于 12 宫格二级菜单（更多）内，或作为全局设置（齿轮）的首选开关。

### 3.6 心形眼神光视觉规格 (Heart Catchlight Visual Spec) [NEW]
基于参考设计图分析，心形视觉须达到以下标准（当前实现与之存在显著差距）：
- **形体尺寸**：心形有效宽度须达到屏幕宽度的 **80%**，当前约为 55%，需扩大。
- **填充方式**：内部必须有大面积**粉色实心填充**（而非空心轮廓），使用径向渐变从粉白色过渡到玫瑰红。
- **发光效果**：外发光模糊半径须达到 sigma >= 30，产生类似真实补光灯的大范围环境漫反射。
- **背景光晕**：屏幕底部应有一个半圆弧形的玫瑰红环境光晕，作为心形的背景烘托层。

### 3.7 相机与硬件集成规格 (Camera & Hardware Integration Spec) [NEW]
**实施状态**: ✅ 已完成（里程碑 3，2026-04-13）

#### 3.7.1 相机流挂载 (Camera Stream Integration)
- **初始化流程**:
  1. 应用启动后延迟 1.5s 检查权限
  2. 权限授予后自动调用 `availableCameras()`
  3. 创建 `CameraController` 实例（后置摄像头，高分辨率）
  4. 异步初始化完成后触发 UI 重建
  5. 将控制器传递给 `PipContainer` 显示实时预览

- **技术参数**:
  - **分辨率**: `ResolutionPreset.high` (1080p+)
  - **音频**: 禁用 (`enableAudio: false`)
  - **帧率目标**: ≥ 30fps
  - **初始化超时**: 5s

- **PIP 预览规格**:
  - **尺寸**: 120x160 (3:4 竖屏比例)
  - **圆角**: 12dp
  - **边框**: 粉色虚线描边（1.2px，6px 实线 + 4px 间隔）
  - **占位状态**: 猫咪背景（12% 透明度）+ "MIAO CAM" 文字

- **交互功能**:
  - ✅ 拖拽移动：实时跟随手指
  - ✅ 四角吸附：松手后自动吸附到最近角落（420ms 弹性动画）
  - ⚪ 缩放手势：双指缩放（待实现，P2 优先级）

- **资源管理**:
  - 页面退出时自动释放相机资源
  - 应用进入后台时可选暂停相机流（节省电量）
  - 应用恢复时自动重新初始化

#### 3.7.2 亮度控制系统 (Brightness Control System)
- **亮度锁定逻辑**:
  - 进入补光页面自动设置屏幕亮度为 100%
  - 使用 `screen_brightness` 插件调用系统 API
  - 异常处理：不支持的设备不会崩溃，仅输出日志

- **亮度恢复逻辑**:
  - 页面退出时调用 `resetScreenBrightness()` 恢复用户原始设置
  - 确保不影响用户其他应用的使用体验

- **技术实现**:
  ```dart
  // 启用亮度锁定
  await ScreenBrightness().setScreenBrightness(1.0);
  
  // 恢复原始亮度
  await ScreenBrightness().resetScreenBrightness();
  ```

#### 3.7.3 屏幕常亮系统 (WakeLock System)
- **常亮逻辑**:
  - 进入补光页面自动启用 WakeLock
  - 防止拍摄过程中屏幕自动熄灭
  - 使用 `wakelock_plus` 插件跨平台支持

- **释放逻辑**:
  - 页面退出时自动释放 WakeLock
  - 避免不必要的电量消耗

- **技术实现**:
  ```dart
  // 启用屏幕常亮
  WakelockPlus.enable();
  
  // 释放屏幕常亮
  WakelockPlus.disable();
  ```

#### 3.7.4 权限管理流程 (Permission Management Flow)
- **权限类型**:
  - `CAMERA`: 相机访问权限（必需）
  - `READ_MEDIA_IMAGES`: 读取图片权限（Android 13+）
  - `READ_MEDIA_VIDEO`: 读取视频权限（Android 13+）
  - `WAKE_LOCK`: 屏幕常亮权限（自动授予）

- **请求流程**:
  1. 应用启动后延迟 1.5s（确保首帧渲染完成）
  2. 检查相机和存储权限状态
  3. 未授予时弹出系统权限请求对话框
  4. 用户授予后自动初始化相机
  5. 用户拒绝时显示友好提示
  6. 永久拒绝时引导用户跳转系统设置

- **用户体验优化**:
  - 权限授予后显示 SnackBar："✨ 补光引擎已就绪"
  - 权限拒绝时显示对话框，提供"去设置"按钮
  - 不强制要求权限，允许用户先体验 UI

#### 3.7.5 错误处理与降级策略 (Error Handling & Fallback)
- **相机初始化失败**:
  - 捕获 `CameraException` 并输出调试日志
  - PIP 容器显示占位状态，不影响其他功能
  - 用户可以继续使用补光功能（无相机预览）

- **亮度控制失败**:
  - 捕获异常并输出日志
  - 不阻塞应用启动
  - 用户可以手动调节系统亮度

- **设备兼容性**:
  - 无相机设备：显示占位符，禁用拍照功能
  - 低端设备：自动降级分辨率（待实现）
  - 不支持亮度 API 的设备：静默失败，不影响使用
