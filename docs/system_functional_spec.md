# 《喵喵补光灯》系统功能规格说明书 (Detailed v2.1)

## 1. 产品概述
《喵喵补光灯》(MiaoMiao Fill Light) 是一款专为暗光摄影设计的移动辅助应用。通过将手机屏幕转化为可控的高亮度光源，并结合独特眼神光映射技术，为用户提供便携式的“专业影棚级”补光体验。

## 2. 核心功能模块详细说明 (12 宫格快捷工具)

基于 UI 截图中的 4x3 功能矩阵，系统需实现以下 12 个核心工具模块：

### 2.1 补光基础 (Lighting Basics)
1. **全屏 (Full Screen)**: 
   - **逻辑**: 一键隐藏所有 UI 控件（除返回手势外），使屏幕以设定的色温/颜色进行全覆盖发光。
2. **色卡 (Color Card)**: 
   - **交互**: 唤起底部滑动手册，支持快速切换 11+ 类摄影色温预设（冷白、暖阳、氛围蓝等）。
3. **亮度 (Brightness)**: 
   - **逻辑**: 直接调用硬件 API 调节屏幕背光，支持在 App 内通过垂直滑动即时微调亮度。

### 2.2 拍摄参数 (Shooting Parameters)
4. **比例 (Ratio)**: 
   - **选项**: 支持 9:16 (全面屏)、3:4 (经典人像)、1:1 (正方形社交图)。
   - **实现**: 动态调整 `CameraPreview` 的 `AspectRatio`。
5. **高清 (HD)**: 
   - **逻辑**: 切换预览分辨率（720P/1080P/4K），优化样片导出质量，支持高码率录制。
6. **缩放 (Zoom)**: 
   - **逻辑**: 支持单指滑动或双指捏合变焦，UI 实时显示变焦倍数（1x - 5x）。

### 2.3 拍摄模式 (Capture Modes)
7. **单拍 (Single Shot)**: 
   - **逻辑**: 默认快门模式，点击即抓拍当前画面。
8. **延迟 (Delay)**: 
   - **配置**: 提供 3s/5s/10s 倒计时拍摄。屏幕中央需显示巨大的倒计时跳字。
9. **自拍 (Selfie)**: 
   - **功能**: 切换前后摄像头，并默认开启暗光实时增强算法。

### 2.4 辅助工具 (Auxiliary Tools)
10. **音效 (Sound Effect)**: 
    - **逻辑**: 快门音效开关，静音模式下拍摄不发出物理快门声。
11. **爱心眼 (Heart Eyes)**: 
    - **原理**: 在黑场中央投影淡粉色心形轮廓。
    - **效果**: 通过光影反射在瞳孔中刻画出“心形眼神光”，并支持调节形状粗细。
12. **更多 (More)**: 
    - **扩展**: 二级入口，包含网格参考线、镜像翻转、测平仪及样片库管理。
13. **语言切换 (Language Switch)**: [NEW]
    - **入口**: 位于主菜单的“更多”设置（More）中。
    - **功能**: 实现全局界面的中英文双语一键切换。
13. **语言切换 (Language Switch)** [NEW]:
    - **功能**: 中英文一键切换 (CN/EN)。
    - **位置**: 位于 12 宫格二级菜单（更多）内，或作为全局设置（齿轮）的首选开关。

## 3. 补光引擎高级逻辑 (Smart Lighting Engine)
- **色域控制**: 支持全 RGB 色值精准调节，锁定系统亮度至 100%。
- **硬件压榨**: 强制请求系统 WakeLock，防止拍摄中熄屏。

## 4. AI 美颜与 VIP 滤镜 (AI Beauty & VIP)
- **算法矩阵**: - **磨皮/美白 (Pending)**: 第一阶段侧重界面实现，算法逻辑待定。
- **VIP 验证**: 开发阶段实行**代码级全量解锁**，方便功能完整性测试，商业化逻辑预留占位符。

### 2.4 系统逻辑与异常处理
- **权限管理**: 透明化的摄像头及写入权限引导流程。
### 3.3 国际化支持 (I18n) [NEW]
- **功能逻辑**: 使用 `flutter_localizations` 插件实现动态语种切换。
### 3.4 启动与权限引导 (Onboarding)
- **引导页逻辑**: [UPDATED] 为了提升用户留存与沉浸感，首屏现改为“品牌自动展示页”。
    - **自动跳转**: 进入页面后展示品牌 Slogan，3 秒后自动通过透明渐变（Fade Transition）跳转至主拍摄界面，无需用户手动操作。
    - **静默授权**: 初始阶段不再强制弹出授权窗口，将权力交还给品牌初次体验。
- **动态权限申请**: [UPDATED]
    - 进入主界面 (`LightingPage`) 时，系统静默检测当前权限。
    - 若检查到核心权限缺失，系统会在主界面自动触发权限申请逻辑，并提供平滑的动态反馈。
- **UI 渲染与可见度优化记录 [NEW]**:
    - **眼神光心向位移修正**: 精确计算 `HeartPainter` 的重心位置，确保心形图案在不同长宽比机型下均能保持中心锚定。
    - **暗光环境可见度补强**: 针对部分暗光机型显示效果不佳的问题，增强了背景装饰文字的对比度（Opacity 升至 0.12+），并为 PIP 预览窗添加了常驻的发光扩散描边，确保在全黑环境下功能入口清晰可见。

### 3.5 底部操作区功能规格 (Bottom Action Zone Spec) [NEW]
基于参考设计图 (`ui_references/main_screen.png`) 对比分析，主界面底部区域需实现以下两个完整功能层：

**快捷工具行 (Shortcut Toolbar)**
- 紧贴 PIP 预览窗下方，贯穿屏幕宽度。
- 包含以下 4 个功能节点（左→右）：
  1. `Beauty`（美颜）：点击进入美颜强度调节面板。
  2. **快门按钮**（核心操作）：64dp 粉色实心圆背景，相机图标居中，单击触发拍照、长按触发视频录制。
  3. `Filter`（滤镜）：点击进入滤镜预设选择。
  4. `Timer`（定时）：点击循环切换 关/3s/5s/10s 倒计时状态。

**底部主导航栏 (Bottom Tab Bar)**
- 采用半透明毛玻璃胶囊容器，固定于屏幕底部 SafeArea 之上。
- 4 个导航节点：
  1. `Home` — 返回欢迎//设置页。
  2. `Fill Light` — **当前主页（默认选中）**，图标为粉色心形，带底部粉色高亮条。
  3. `Filters` — 进入 AI 滤镜与风格化色卡浏览页。
  4. `Gallery` — 进入本地样片库浏览与管理页。

### 3.6 心形眼神光视觉规格 (Heart Catchlight Visual Spec) [NEW]
基于参考设计图分析，心形视觉须达到以下标准（当前实现与之存在显著差距）：
- **形体尺寸**：心形有效宽度须达到屏幕宽度的 **80%**，当前约为 55%，需扩大。
- **填充方式**：内部必须有大面积**粉色实心填充**（而非空心轮廓），使用径向渐变从粉白色过渡到玫瑰红。
- **发光效果**：外发光模糊半径须达到 sigma >= 30，产生类似真实补光灯的大范围环境漫反射。
- **背景光晕**：屏幕底部应有一个半圆弧形的玫瑰红环境光晕，作为心形的背景烘托层。


## 4. 高级扩展功能 (Phase 2 & 3)

### 4.1 商业化与 VIP 系统 (Commercialization)
- **付费模式**: 支持订阅制 (Subscription) 及终身买断制 (Life-time Purchase)。
- **支付集成**: 接入 Apple IAP (内购) 与 Google Play Billing。
- **功能锁逻辑**: 
    - 基础色温与普通相机功能免费。
    - 高级眼神光形状、VIP 艺术滤镜及 4K 高清导出需验证 VIP 权限。

### 4.2 滤镜算法引擎 (Filter Engine)
- **美颜矩阵**: 实现高精度的磨皮、大眼、瘦脸级肤色美化。
- **风格化滤镜**: 
    - **Style**: 日系、清冷、港风。
    - **Artistic**: 撕拉片、赛博朋克、电影感。
- **调节逻辑**: 支持实时调节滤镜强度及颗粒感。

### 4.3 后端与云服务 (Backend & Cloud)
- **账户系统**: 支持手机号/第三方社交账号登录，实现跨设备同步。
- **云端样片库**: 样片自动备份至私有云，支持多端查看与管理。
- **后台管理系统 (Admin)**: 
    - 运营系统：动态更新滤镜色卡配置。
    - 财务系统：查看订阅流水与财务报表。
    - 用户管理：处理异常反馈与申诉。

### 3.5 交付策略 [NEW]
- **阶段一**: UI 框架实现。重点在于：
    - 品牌权限引导页。
    - 主拍摄界面（含 PIP 预览与心形补光）。
    - 12 宫格菜单交互（液态弹出逻辑）。
- **阶段二**: 核心功能注入。重点在于：
    - 相机预览流挂载。
    - 硬件亮度控制逻辑。

## 5. 交互交互与触发逻辑 (Interaction Logic)

### 5.1 12 宫格工具栏触发
根据最终实现的 UI 方案，12 宫格快捷菜单的触发逻辑如下：
- **主要入口**: 位于屏幕**顶部中心偏左**的”↓ 工具”胶囊按钮。
- **交互逻辑**: 点击触发从顶而下的下滑动画，菜单以抽屉形式覆盖屏幕上方区域。
- **隐藏逻辑**: 
    - 点击背景暗化区域或再次点击该胶囊按钮即可收起
    - **[NEW]** 向上滑动手势收起：在菜单任意位置向上拖拽超过 120px 或快速向上滑动（速度 < -600px/s）即可关闭
    - **[NEW]** 拖拽过程中提供实时视觉反馈：手柄宽度和透明度动态变化，菜单位置实时跟随手指
    - **[NEW]** 未达到关闭条件时自动回弹到原位，提供流畅的交互体验
- **布局优化 [NEW]**:
    - 容器高度优化至屏幕高度的 78%，确保 12 个网格项完整显示
    - 内部间距优化：顶部 padding 50px，网格行间距 20px
    - 网格宽高比控制：`childAspectRatio: 0.85`，适配不同屏幕尺寸
    - 无滚动设计：所有工具项直接可见，无需滚动操作

### 5.2 滤镜与设置
- **滤镜切换**: 位于取景框下方的横向滑动托盘，左右滑动实时预览。
- **全局设置**: 位于屏幕右上角的 **齿轮图标**。
- **模式切换**: 位于右上角的 **静态猫头图标**。
